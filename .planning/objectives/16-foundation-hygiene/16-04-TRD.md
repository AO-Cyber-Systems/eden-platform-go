# TRD 16-04 — Config Promotion (alpha → beta)

**Objective:** 16 (Platform Foundation Hygiene)
**Roadmap req:** R26.4
**Status:** planned

## Goal

Promote `platform/config` from alpha to beta with documented conventions and
typed loaders so each consumer can stop hand-rolling env-var parsing.

## Tasks

1. Add typed env loaders (backward-compatible additions):
   - `GetInt(key string, fallback int) int`
   - `GetBool(key string, fallback bool) bool`
   - `GetDuration(key string, fallback time.Duration) time.Duration`
   - `MustGet(key string) string` — panics if unset (boot-time config).
   - `Required(keys ...string) error` — returns aggregated missing-vars error.
2. Multi-environment overrides:
   - `LoadFor(env string) *PlatformConfig` reads from `EDEN_ENV` (or argument)
     and applies any `key__<env>` override (e.g. `DATABASE_URL__prod`).
3. Secret loading:
   - Existing `_FILE` convention preserved.
   - New `_BASE64` convention: if `<KEY>_BASE64` is set, decoded value used.
4. Validation:
   - `(*PlatformConfig).Validate() error` — returns aggregated error listing
     any missing required fields (DatabaseURL is required; others optional with
     reasonable defaults).
5. Document conventions in `platform/config/README.md`:
   - Required vs optional vars with defaults table
   - Secrets handling (`_FILE`, `_BASE64`)
   - Multi-environment overrides (`__<env>` suffix)
6. Tests: cover all new exports.

## Files

- `platform/config/config.go` (extend) — typed loaders, Validate, LoadFor
- `platform/config/secrets.go` (new) — `_BASE64` secret handler
- `platform/config/README.md` (new)
- `platform/config/config_test.go` (extend) — typed loaders, Validate, LoadFor
- `platform/config/secrets_test.go` (new)
