# TRD 16-01 — Audit Pipeline Unification

**Objective:** 16 (Platform Foundation Hygiene)
**Roadmap req:** R26.1
**Status:** in-progress

## Goal

Extend `platform/audit` (the canonical implementation) so the patterns we already
see scattered across `aosentry/audit`, `eden-biz/audit`, and `audithelper` are
covered by platform. No consumer migration in this objective — only the platform
package becomes the single source of truth that future migrations can target.

## Tasks

1. Add a public schema documented in `platform/audit/README.md`:
   - `Event` shape (existing) with explicit field semantics
   - Action naming convention: `domain.entity.verb` (e.g. `auth.user.login`)
   - Standard `Details` keys (`before`, `after`, `reason`, `request_id`)
2. Extend the package surface in a backward-compatible way:
   - `Action` typed-string family with the most-used canonical action constants
     (login/logout/signup/create/update/delete/grant/revoke).
   - `EventFromHTTP(r *http.Request) Event` helper that pulls IP + user-agent +
     request_id into the Event. Consumers fill the rest.
   - `LogSync(ctx, e) error` for synchronous fallback (used by tests and code
     paths that need a hard guarantee the row was written).
   - `WithDetails(k, v)` helper on `Event` that builds the details map fluently.
3. Update tests: cover all new exports. Existing tests stay green.

## Files

- `platform/audit/event.go` (new) — Event helpers, Action constants, EventFromHTTP
- `platform/audit/logger.go` (extend) — add `LogSync`
- `platform/audit/README.md` (new) — canonical schema doc
- `platform/audit/event_test.go` (new) — coverage
- `platform/audit/logger_test.go` (extend) — LogSync test

## Done when
- `go test ./platform/audit/...` green
- `go vet ./platform/audit/...` green
- README documents schema + Action conventions
